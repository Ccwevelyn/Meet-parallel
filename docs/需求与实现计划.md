## 1. 更换模型（DS v3 / Gemini / GPT-5）

当前项目已支持任意 OpenAI 兼容接口，通过环境变量即可切换：

| 模型 | 建议配置 |
|------|----------|
| **DeepSeek v3** | `AI_API_KEY=sk-xxx`，`AI_BASE_URL=https://api.deepseek.com/v1`，`AI_MODEL=deepseek-chat` 或最新模型名 |
| **Gemini** | 使用 Google AI Studio 的 API Key，需兼容层或直接调用 Google API（见下） |
| **GPT-5 / OpenAI** | `AI_API_KEY=sk-xxx`，`AI_BASE_URL=https://api.openai.com/v1`，`AI_MODEL=gpt-5` 或当前最新名 |

**建议优先购买的 API**：若选其一，**DeepSeek API** 性价比高且已兼容当前代码；若要用 Gemini，需在代码里加一条分支调用 Google 接口。

## 3. RAG 训练（从外网获取信息再参与生成）

目标：从小红书、微博、网页等抓取内容 → 切片、向量化 → 生成时检索并注入上下文，让回复更「像活人」、有信息量。

**建议购买的 API / 服务**：

| 用途 | 推荐 | 说明 |
|------|------|------|
| **向量化 (Embedding)** | OpenAI `text-embedding-3-small` 或 **Cohere Embed** | 将文本转为向量，用于检索；OpenAI 与现有 Key 可复用 |
| **检索/向量库** | 自建用 **Chroma**（本地）或 **Pinecone**（云） | 存向量、按相似度检索；Chroma 无需付费 |

**实现思路**：新增「外部语料」表或文件 → 定时/手动跑脚本：抓取或读取网页/社交文本 → 调用 Embed API → 写入向量库；生成回复前按当前话题或成员检索若干条，拼进 system/user 上下文。代码上可先做「可插拔 RAG 模块」：有 key 则检索并注入，无则跳过。

## 5. 检索网络时事 / 二次元动漫并用于发言

目标：按角色偏好（有人爱分享二次元/游戏）检索热点或动漫相关内容，在生成时作为可选上下文，让 AI 能「分享」这些内容。

**建议购买的 API**：

| 用途 | 推荐 | 说明 |
|------|------|------|
| **通用搜索/热点** | **Serper**（Google 搜索 API）或 **Tavily** | 查时事、动漫、游戏关键词，返回摘要或链接，可拼进 prompt |
| **二次元/动漫资讯** | 若有现成开放 API 或 RSS 可接；否则用 Serper 搜「动漫 新番」等 | 暂无统一必买 API，可先用 Serper 泛搜 |

**实现思路**：  
- 配置 `TRENDS_API_KEY`（如 Serper）+ 可选 `TRENDS_QUERY` 或按成员配置「兴趣标签」。  
- 定时或按需请求一次「热点/动漫」摘要；生成该成员回复时，若匹配兴趣则把摘要注入上下文，并提示模型可自然引用。  
- 代码上可先做「趋势/兴趣上下文」模块：有 key 则拉取并注入，无则跳过。
